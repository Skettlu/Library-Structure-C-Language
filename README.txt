#Library Management System (C)

# Περιγραφή  
Το πρόγραμμα υλοποιεί ένα σύστημα διαχείρισης βιβλιοθήκης σε γλώσσα C, με δυνατότητες για:  
- Διαχείριση κατηγοριών βιβλίων (Genres)  
- Καταχώριση και οργάνωση βιβλίων (Books)  
- Καταχώριση μελών (Members)  
- Δανεισμούς βιβλίων (Loans) και επιστροφές (Returns), με δυνατότητα αξιολόγησης (Reviews)  
- Αναζήτηση βιβλίων με βάση τον τίτλο (BookIndex / AVL)  
- Κατάταξη βιβλίων ανά βαθμολογία και προτάσεις κορυφαίων βιβλίων (RecommendationHeap + TOP)  
- Παρακολούθηση δραστηριότητας μελών (MemberActivity): πόσα δανειστήκαν, πόσες αξιολογήσεις έκαναν  
- Στατιστικά βιβλιοθήκης (stats): συνολικά βιβλία, μέλη, δανεισμοί και μέση βαθμολογία  
- Καθαρισμό και απελευθέρωση όλων των δομών (μνήμη, λίστες, δέντρα, heap) για ασφαλή τερματισμό  

Η εκτέλεση των λειτουργιών γίνεται με εντολές που διαβάζονται από αρχείο εισόδου (`stdin` ή αρχείο `.txt`).

#Εντολές

1. SLOTS:
    -> S <n_Slots>
    Ορίζει τον αριθμό SLOTS για το display (πόσα βιβλία συνολικά θα προβληθούν).

2: G- Genre Insertion:
    -> G <gid> <name>
    Δημιουργεί νέα κατηγορία.
    Αν υπάρχει ήδη το ίδιο gid(genre-id), η εντολή αγνοείται.

3: BK- Book Insertion:
    -> BK <bid> <gid> <title>
    Προσθέτει νέο βιβλίο στη συγκεκριμένη κατηγορία.

    Η λίστα βιβλίων κάθε κατηγορίας διατηρείται ταξινομημένη φθίνουσα κατά avg (μέσος όρος αξιολογήσεων)
    και, σε ισοβαθμία, κατά αύξον bid.

4: M- Member Insertion
    -> M <sid> <name>
    Προσθέτει νέο μέλος στη βιβλιοθήκη, ταξινομημένα κατά sid(student-id).

5: L- Loan To Member
    ->L <sid> <bid>
    Ορίζει ότι το μέλος sid δανείζεται το βιβλίο bid.
    Αν δεν υπάρχει μέλος ή βιβλίο ή ο δανεισμός είναι διπλός → IGNORED.
    Αλλιώς → DONE.

6: R- Return Of Book
    ->R <sid> <bid> <score> <status>
    Επιστρέφει ένα δανεισμένο βιβλίο.
        *Αν status = ok -> ενημερώνει τη βαθμολογία (score∈[0..10]).
        *Αν status = lost -> το βιβλίο μαρκάρεται ως χαμένο.
        *Αν το μέλος ή ο δανεισμός δεν υπάρχουν -> IGNORED.

7: D- Create Display [Βιτρίνα]
    ->D
    Υπολογίζει ποιες κατηγορίες και ποια βιβλία θα εμφανιστούν στο display.

    Κατανέμει τις SLOTS θέσεις προβολής στις κατηγορίες με μέτρο & 
    μεγαλύτερα υπόλοιπα και επιλέγει τους τίτλους που θα προβληθούν 
    ανά κατηγορία.

8: PG- Print Genre
    ->PG <gid>
    Εμφανίζει κάθε βιβλίο της κατηγορίας gid, σε φθίνουσα σειρά
    στην μορφή: <bid>, <avg>

9: PM- Print Member
    ->PM <sid>
    Εμφανίζει όλα τα βιβλία που έχει δανειστεί το μέλος sid [Ενεργών δανεισμών].

10: PD- Prints Display
    ->PD
    Εμφανίζει τις κατηγορίες και τα βιβλία που προβάλλονται στο display:

    Display:
    <gid>:
    <bid>, <avg>
    ...

Νέες Εντολές (Phase 2)
-----------------------
11: F <title>
    - Αναζήτηση βιβλίου κατά τίτλο.
    - Εκτυπώνει: Book <bid> "<title>" avg=<avg>
    - Αν δεν βρεθεί: NOT FOUND
    - Πολυπλοκότητα: O(log n) με το AVL (global ή per-genre).

12: TOP <k>
    - Εμφανίζει τα k κορυφαία βιβλία από τον RecommendationHeap.
    - Έξοδος:
        Top Books:
        <bid> "<title>" avg=<avg>
        ...
    - Σημείωση για implementation:
      * Η προτεινόμενη υλοποίηση χρησιμοποιεί το RecHeap (capacity = 64) — τότε η
        εντολή είναι πολύ γρήγορη (σταθερή ανώτατη πολυπλοκότητα).
      * Εναλλακτικά, αν απαιτείται να σαρώνει όλα τα βιβλία, η πολυπλοκότητα γίνεται O(n + k log n).
    - Lost βιβλία (lost_flag=1) πρέπει να είναι αφαιρεμένα από το heap.

13: AM
    - Εκτυπώνει τα πιο ενεργά μέλη (loans_count + reviews_count > 0).
    - Μορφή εξόδου:
        Active Members:
        <sid> <name> loans=<L> reviews=<R>
      Αν δεν υπάρχουν: NO ACTIVE MEMBERS

14: U <bid> "<new_title>"
    - Ενημερώνει τον τίτλο του βιβλίου bid.
    - Έλεγχος μοναδικότητας τίτλου (global AVL) -> αν υπάρχει ήδη: IGNORED
    - Διαγραφή από AVL (παλιό title) + επανεισαγωγή με νέο title.
    - Έξοδος: DONE ή IGNORED.
    - Πολυπλοκότητα: O(log n) (delete + insert).

15: X
    - Εμφανίζει System Stats:
      Stats: books=<#> members=<#> loans=<#> avg_rating=<avg>
    - Σημείωση: το πεδίο "loans" μπορεί να αντιπροσωπεύει τα συνολικά ενεργά δάνεια
      (προτείνεται: sum των MemberActivity.loans_count). Το avg_rating είναι ο συνολικός μέσος όλων
      των αξιολογήσεων = total_score_sum / total_reviews (φορμάρισμα με 2 δεκαδικά).

16: BF
    - Απελευθερώνει όλη τη δυναμική μνήμη:
      genres, per-genre AVL nodes, global AVL (αν υπάρχει), βιβλία, μέλη, loans,
      MemberActivity list, RecHeap.
    - Εκτύπωση: DONE
